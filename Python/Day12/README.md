## 🧠 Interview Preparation


### Q1. 1. What is OOP?
Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around data (objects) rather than functions or pure logic. It structures a program by bundling related properties and behaviors into self-contained units.
### 2. What is a Class?
A Class is an abstract blueprint, template, or schema used to define the structure of an entity. It specifies what data fields and actions will exist, but it does not occupy memory space for data storage itself.
### 3. What is an Object?
An Object is a concrete, live instance of a class built directly from its blueprint. It occupies physical space in system memory ($RAM$) and holds real, independent data values.
### 4. What is a Constructor?
A Constructor (written as __init__ in Python) is a special lifecycle method that executes automatically the exact millisecond a new object is instantiated. Its primary job is to allocate initial values to the object's attributes.
### 5. What is self?
self is a keyword parameter that represents the specific object instance currently executing the code. Because a class blueprint is shared, Python uses self as an internal pointer to ensure it updates or reads the variables of the exact object you targeted, preventing data corruption across instances.
### 6. Difference between Class and Object?
A Class is the design map (e.g., a blueprinted schematic for a smartphone). It handles no active state and exists only in the source code.An Object is the actual physical product (e.g., a real smartphone sitting on your desk). It handles live data state changes dynamically at runtime.
### 7. What are Attributes?
Attributes are variables bounded inside an object that track its internal data state or characteristics (e.g., self.balance = 500.00 or self.is_checked_out = True).
### 8. What are Methods?
Methods are functions defined inside a class that govern what an object can do (its behaviors). They are typically used to securely calculate metrics, format information, or mutate internal attributes.
### 9. Why do we use OOP?
Encapsulation: It locks data and code together inside a protective wall, blocking external scripts from introducing corrupt values.Code Reusability: It allows you to write a clean blueprint once and instantly instantiate millions of unique records.Maintainability: If a business rule changes, you update a single method inside the class framework rather than tracking down thousands of isolated variables across your codebase.
### 10. Where is OOP used in real software?
E-Commerce Engines: A master Product class dictates how catalog listings calculate bulk discounts, track warehouse inventories, and update shopping cart vectors.GUI Frameworks: Every digital window, button, and slider item you click on an operating system or app is an instance object inheriting from a core Widget class.Video Game Engines: Game loops track characters (players, enemies, projectiles) as individual objects managing real-time coordinates, health values, and rendering updates.
